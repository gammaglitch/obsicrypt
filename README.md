# Obsicrypt

An Obsidian plugin that encrypts sensitive content inside notes with a master password.

This plugin was **one-shot generated by Claude** from a single prompt describing the feature (see `docs/log.md` for the build log). Everything — crypto, UI, settings, tests, docs — was produced in one session.

## What it does

- You set a master password once in Settings → Obsicrypt.
- In the command palette, **Obsicrypt: Encrypt selection** wraps the current selection in a `` ```secret `` fenced block and encrypts its contents.
- In reading view, secret blocks appear locked. Click **Unlock** to enter your password and reveal the plaintext.
- **Obsicrypt: Lock vault** clears the cached password from memory.

## How the encryption works

- AES-GCM-256 for encryption, per-secret random salt (16 B) and IV (12 B).
- Key derived from the master password via PBKDF2-SHA256 with 600 000 iterations.
- Ciphertext lives inline in the note as `obsicrypt:v1|<salt>|<iv>|<ciphertext>` (base64-encoded segments), so the note remains self-contained.
- Only a verifier (AES-GCM of a fixed plaintext) is persisted via `plugin.saveData` — your master password is never written to disk.

## Building

```bash
pnpm install
pnpm build
```

Set `OBSIDIAN_PATH` in `.env` to point at your dev vault and the build emits straight into `<vault>/.obsidian/plugins/obsicrypt/`.

```bash
pnpm test        # crypto + envelope unit tests
pnpm build:ts    # type check
```
